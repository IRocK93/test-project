## Implementation Notes

### Badge Definition API

Badge definitions are loaded via `getBadgeDefinitions()` which calls `GET /api/badges/definitions`. Each definition should include:
```json
{
  "badgeType": "T01",
  "name": "First Giggle",
  "description": "Log 2 milestones with type SOCIAL or PLAY",
  "tier": "BRONZE",
  "xpValue": 10,
  "category": "traits"
}
```

### Trait-Based Unlock Logic

Trait badges are evaluated against:
1. The BabyMon's current `traits: String[]` field in Prisma
2. Entry counts by type/category
3. Streak tracking (consecutive days)
4. Trait activation duration (days since trait was set)

### Backend Requirements

To support trait-based badge evaluation, the backend service should:
1. Query the BabyMon's `traits` array
2. Count entries matching badge requirements
3. Auto-award Bronze badges when criteria are met
4. Track trait activation timestamps for Gold badge progression requirements
